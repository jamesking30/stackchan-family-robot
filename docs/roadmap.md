# 开发路线

## M0 — 本地控制平面（已完成）

- 角色版本、回答方式热更新和回滚。
- 四用户模型、记忆隔离、儿童审核。
- Codex/OpenClaw 状态显示接口。
- 管理网页、CLI、MCP 与自动测试。

验收：本机启动后可修改回答方式，重启不丢失；用户 A 的记忆不能由用户 B 查询或删除；任务进度可转换成显示状态。

## M1 — 真机通信与表情动作（当前）

- 识别 StackChan 主控板型、USB 串口和舵机配置。
- 完整备份 1.4.4 固件与设备设置。
- 构建 AIAvatarStackChan 固件分支，加入官方 BSP 所需驱动。
- 打通 Mac 到真机的 WebSocket、文字、表情、头部动作和心跳。

验收：断网安全、重连恢复、表情/口型同步；原 1.4.4 可恢复。

进度：产品固件已写入非活动 `ota_0`，启动和硬件初始化、设备鉴权、WebSocket 心跳、文字、表情、低幅动作、回中、Codex 任务状态和断线自动重连均已通过真机验证。原厂 `ota_1`、NVS、Bootloader 与分区表均保留，并有两份校验一致的完整备份。完整原厂恢复演练留到维护窗口执行；刷写记录见 `docs/m1-flash-validation-2026-07-19.md`。

## M2 — 中英双语语音

- AIAvatarKit 实时网关、VAD、可打断播放。
- OpenAI 语音识别/对话/语音合成；可选本地唤醒词与 VAD。
- 访客会话、语言自动切换、延迟指标。

验收：安静家庭网络中首个语音响应目标 < 1.5 秒，连续中英切换和打断稳定。

## M3 — 四人身份与记忆

- 本地人脸登记、活体/置信度、访客降级。
- 记忆从 SQLite 迁移到 ChatMemory + PostgreSQL/pgvector。
- 成人审核、查询、纠正、遗忘和数据导出。

验收：四人交叉测试无串记忆；识别不确定时不写入个人长期记忆。

## M4 — 家庭设备与代理任务

- OpenClaw 接入 Home Assistant；按风险分层确认。
- Codex/OpenClaw 自动上报排队、执行、等待、完成、失败状态。
- 真机状态卡、完成提醒和失败解释。

验收：低风险设备可控；门锁、摄像头、购买、账户操作必须由已登记成人确认。

## M5 — 长期在线与维护

- macOS LaunchAgent、自启动、健康检查、日志轮转和数据库备份。
- 固件 OTA、灰度发布、遥测和一键恢复。
- 对话质量、安全与记忆回归评估。
