# StackChan 真机检查记录 — 2026-07-19

## 结论

设备是 M5Stack 官方一体式 StackChan，固件板型标识为 `m5stack-stack-chan`，不是旧款 Core2/Atom 拼装路线。官方 `m5stack/StackChan` 仓库提交 `b72b3ede` 中存在完全对应的板级实现和相同分区表，可作为固件主基线。

## 已确认

- ESP32-S3 QFN56 revision v0.2，双核 240MHz。
- 16MB QIO Flash，80MHz；8MB PSRAM，80MHz。
- 原生 USB Serial/JTAG，macOS 端口 `/dev/cu.usbmodem101`。
- 固件项目 `stack-chan`，版本 `1.4.4`，ESP-IDF `5.5.4`。
- 活动镜像位于 `ota_1`，偏移 `0x510000`。
- 320×240 ILI9342 兼容屏、FT6336 触摸、GC0308 摄像头。
- AXP2101 电源管理、AW9523/PY32 扩展 IO。
- AW88298 扬声器与 ES7210 麦克风音频链路，24kHz。
- 两个 SCS 总线舵机，设备 ID 为 1 和 2。
- Secure Boot 与 Flash Encryption 均未启用。

## Flash 分区

| 分区 | 偏移 | 大小 | 用途 |
|---|---:|---:|---|
| `nvs` | `0x00009000` | `0x00004000` | 本机配置，私密 |
| `otadata` | `0x0000d000` | `0x00002000` | OTA 选择 |
| `phy_init` | `0x0000f000` | `0x00001000` | 射频初始化 |
| `ota_0` | `0x00020000` | `0x004f0000` | OTA 应用 A |
| `ota_1` | `0x00510000` | `0x004f0000` | OTA 应用 B，当前活动 |
| `assets` | `0x00a00000` | `0x00400000` | 表情/资源数据 |
| `coredump` | `0x00e00000` | `0x00010000` | 崩溃转储 |

## 备份验证

- 完整读取范围：`0x00000000–0x00ffffff`。
- 文件大小：`16,777,216` 字节。
- SHA-256：`fec70055089ecfe29c109ac9893824677d529d4425b26eae4ecb7042af7a522a`。
- 使用不同波特率完成第二次完整读取；SHA-256 相同，逐字节比较一致。
- 备份留在 Git 忽略目录，并复制到用户本机的私有备份目录。

## 固件决策

保留官方 ESP-IDF 5.5.4、板级 HAL、分区布局和 OTA 能力。在此基础上接入 Mac 实时对话网关；不使用面向旧 Core2 的固件直接覆盖设备。第一轮真机开发优先通过 OTA/非活动分区验证，保留 `ota_1` 的 1.4.4 恢复路径。
