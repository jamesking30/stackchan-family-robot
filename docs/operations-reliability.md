# StackChan 长久在线方案

机器人由 Mac 上的控制服务、主语音 GPT-SoVITS、备用语音 Qwen 和机器人固件共同组成。

## 已实施

- `run_robot_services.sh` 同时管理控制服务和两套本地语音；语音进程退出后会自动重启。
- `install_launch_agent.sh` 将服务注册为 macOS 登录后自启，并在异常退出后重新拉起。
- Mac 接通电源时，后台服务会阻止主机因空闲而睡眠；合盖睡眠仍需在系统设置中单独处理。
- `check_robot_health.py` 检查控制服务、两套语音、机器人在线状态，以及 Mac 地址是否与固件配置一致。
- GPT-SoVITS 不可用时自动切换到 Qwen，再失败才使用 macOS 系统语音。
- GPT-SoVITS 启动后会自动预热模型和参考音频缓存，避免第一次对话额外等待。

## 网络地址必须固定

当前固件保存的是 Mac 网关地址。家用路由器通过 DHCP 改变 Mac 地址后，机器人不会知道新地址，重启服务也不能修复。

在路由器中建立 DHCP 地址保留：

- Mac Wi-Fi 地址：`5c:9b:a6:82:a2:dc`
- 保留地址：`192.168.31.174`
- 服务端口：`8765`

保留完成并让 Mac 重新获得该地址后，机器人会自动重连。不要同时把这个地址分配给其他设备。

后续固件版本应增加 mDNS 自动发现及 NVS 网关地址更新，以彻底消除对固定 IP 的依赖；在该固件完成真机验证之前，路由器地址保留是风险最低的生产方案。

## 日常检查

运行：

```bash
./scripts/check_robot_health.py
```

最近一次检查会保存到 `var/health/latest.json`，服务日志位于 `var/logs/`。
